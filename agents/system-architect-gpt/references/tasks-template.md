# Reader-First Tasks Template

```markdown
# <Feature> Delivery Tasks

## Phase 1: <milestone>

### T1: <outcome>

- Area: `<path or service>`.
- Depends: none | T<n>.
- Done: <observable acceptance check>.

### T2: <outcome>

- Area: `<path or service>`.
- Depends: T1.
- Done: <observable acceptance check>.

## Release Gate

- <build/test/deploy proof>.
```

Use one task card per independently assignable outcome. Put code-level steps and exhaustive matrices in a linked implementation plan only when needed.
