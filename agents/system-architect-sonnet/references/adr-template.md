# Reader-First ADR Template

```markdown
# ADR: <Decision>

Status: Proposed | Accepted

## At a Glance

- Outcome: <one sentence>
- Scope: <included>; excludes <important exclusion>.
- Chosen: <one sentence>.

## Decisions

### D1: <decision>

- Why: <one reason>.
- Consequence: <main trade-off or invariant>.

### D2: <decision>

- Why: <one reason>.
- Consequence: <main trade-off or invariant>.

## Boundaries

- `<owner>` owns <responsibility>; does not own <responsibility>.
- `<dependency>` direction: <from> -> <to>.

## Delivery Constraints

- Rollout: <only when changed>.
- Risk: <highest risk> -> <specific mitigation>.
- Verify: <one observable proof>.

## References

- `<path or URL>` - <why it matters>.
```

Omit empty sections. Keep full alternatives, evidence, and scoring in reasoning; cite only decision-relevant sources.
